# web-site-archive

本仓库用于归档和发布多个独立的 HTML 页面。所有页面均为静态网页，不依赖特定域名或后端服务，可直接在浏览器中打开或通过静态托管服务访问。

## 项目简介

web-site-archive 是一个通用的 HTML 页面归档仓库。你可以将各种独立的网页资源（如信息页面、工具页面、临时展示页等）存放在此，并通过 Git 进行版本管理。仓库本身不限定任何特定主题或用途，仅提供结构化的存储与发布能力。

## 目录说明

```
web-site-archive/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 公共资源文件（样式、脚本、图片等）
├── index.html      # 可选：仓库首页或导航页
└── README.md       # 本文件
```

- **pages/**：每个独立页面可放在子目录中，也可直接存放。建议按功能或时间命名文件夹。
- **assets/**：存放多个页面共享的 CSS、JavaScript、字体或图像等资源。
- **index.html**：若需要，可编写一个简单的导航页，列出所有归档页面。

## 页面归档说明

1. 所有页面应为完整的 HTML 文件，包含必要的 `<!DOCTYPE html>` 声明。
2. 页面资源引用请使用相对路径，以确保在本地或不同托管环境下均可正常访问。
3. 如需引用公共资源，建议统一放置在 `assets/` 目录下。
4. 每个页面应保持独立，避免跨页面依赖（除非通过导航链接）。

## 维护说明

- 仓库采用 Git 进行版本管理，每次新增或修改页面请提交清晰的 commit 信息。
- 建议定期清理不再需要的旧版页面，保持仓库整洁。
- 若页面数量较多，可考虑在 `index.html` 中增加搜索或分类功能，方便浏览。
- 本仓库不强制要求任何构建工具或自动化流程，所有页面均为原始 HTML 文件。

## 许可

本仓库内容按需使用，无特定许可证约束。贡献者请自行确保所提交页面不涉及版权或敏感信息。
